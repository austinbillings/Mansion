This is a <box>

A Mansion-specific HTML tag you use to specify, well, a layout box. Of course, <div>s are still totally cool. You can use <div class="box"> for the same effect.

In fact, for every hipster-looking flow:around, xs:12, offset-md:3, and <stack wrap> you'll see, there are perfectly sane-looking CSS classes you can use instead. Namely, 'flow-around', 'box-xs-12', 'offset-md-3',

As the developer, you choose between class-based use: 


  <div class="row wrap">
    <div class="box-xs-12 box-md-4 box-md-offset-2">
      <!-- content -->
    </div>
  </div>
and element/attribute-based use: 

  <row wrap>
    <box xs:12 md:4 offset-md:2>
      <!-- content -->
    </box>
  </row>
Element/attribute usage has several advantages-- more concise, semantic syntax that's easier to read and write. However, you may be writing JSX syntax (which doesn't recognize Mansion attributes, and only partially with JSX's is attribute), have strict institutional markup standards, or need to support older browsers. In these cases, using the classes over the elements/attributes is recommended.

Each <box> tag will flex to fill the space available in its container. The box elements will fill in alongside each other,
